# Faculty Management System

I built this Faculty Management System using Laravel to handle faculty registrations, experience tracking, and administrative management cleanly and efficiently.

## What this project does

This system acts as a dedicated portal for an institution to manage their faculty members. It handles everything from onboarding new teachers to giving admins a birds-eye view of everyone's data.

**Key Features:**
* **Role-Based Access Control:** Distinct login flows and dashboards for `Admin` and `Faculty` users.
* **Smart Registration Form:** An interactive, AJAX-powered form where users can dynamically add or remove multiple "Education" and "Experience" rows. 
* **Fresher Logic:** If a user checks "I am a Fresher", the experience fields automatically hide and drop their required status.
* **Current Job Logic:** Checking "Currently Working Here" disables the end-date requirement for that specific role.
* **Admin Dashboard:** A fully integrated jQuery DataTables interface displaying all registered faculty members.
* **Admin Superpowers:** Admins can view full detailed profiles, edit any faculty member's information, toggle their account status (Active/Inactive), and perform soft-deletes.
* **Custom Styling:** The core views use custom inline CSS and vanilla JavaScript for interactivity, keeping the UI clean without relying heavily on compiling framework stylesheets.

## How to run it locally

If you are pulling this project down to test it, follow these steps to get it running on your machine:

**1. Install Dependencies**

composer install
npm install


**2. Setup the Environment**
Copy the example environment file and generate your application key:

cp .env.example .env
php artisan key:generate


**3. Configure the Database**
Open the `.env` file and update your database credentials to match your local setup:

env

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=faculty_system
DB_USERNAME=root
DB_PASSWORD=



**4. Migrate and Seed**
This will create all the necessary tables and generate the default Admin account:


php artisan migrate:fresh --seed



**5. Link the Storage**
This step is required so the profile photos render correctly on the frontend:

php artisan storage:link



**6. Boot up the server**


php artisan serve
